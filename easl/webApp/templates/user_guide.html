User Guide

This is the eASL User Guide. Since eASL has multiple Users, there will be multiple sections of this User Guide.


Section 1: The Teacher
Section 2: The Parent
Section 3: The Student
Section 4: HowTo



Section 1:
The Teacher: The Teacher User should be used in conjunction with the Student User so
that the Teacher may give the Student the application to relay their actions to the Teacher.
The Teacher User will be able to:
Adding and Removing Students
Adding and Removing Parents
View the Timeline of individual Students
View the Action Log of individual Students
View the Action Log of all Students
Edit Student Information
Edit Parent Information
These actions all fall within the scope of eASL's application.

Section 2:
The Parent: The Parent User will only have access to this application if the Teacher User gives access.
The Parent User will be able to:
Check the daily actions of their child student
View the child's daily action log
View the child's action log over the school year.

Section 3:
The Student: The Student User will have the least functionality regarding actions performed
within the eASL application. The Student User will be given the application via the Teacher
User and will mark the actions the Student wishes to accomplish in the given moment.
The Student User will be able to:
Select action buttons

Section 4:
HowTo: This section describes the basic flow of actions of a regular visit to the
eASL application along with a description for each step.
Step 1: Creating an Account
When the user first opens the eASL application, they will be prompted
to create either a teacher or parent account; depending on what type
of user is using the eASL application.
Step 2: Registering a student
This step is primarily focused on Teacher users because Teachers are the only one
who may add students. After the teacher logs in, they may select "New Student",
which will bring them to the Student Registration Page. The teacher will then input the
Student's first name, last name, Date of Birth, Gender, and an optional parent.
Once all these fields are filled out, the Teacher will click 'Register'
and the new Student will be registered into the database.
Step 3: Logging into the Student's Profile
This step is primarily focused on Teacher users again. Once the Student
is input into the database, the Teacher can click on the dropdown
menu located on the Student Selection Page and navigate to the Student
they just created or an existing Student they are working with and click
'Log in'. From here, the Teacher user will be able to view the Student's Profile Page
along with their timeline, action log, and information about the student.
Step 4: Save Actions
This step may be viewable by all users, but the Student User will be
the one saving the actions. Once the Teacher is on the Student's Profile Page,
the teacher hands the device to the student and the student will be allowed
to select the action that they wish to perform. Once an action is selected,
that action is saved into the database and displayed on the child's timeline as well
as their action log. This action may be viewed by the Teacher and the Parent as well.
